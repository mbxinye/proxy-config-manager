## 改进方案：被墙服务连通性测试 + 智能节点命名

### 核心改动

#### 1. 取消下载测速
- 删除 `scripts/speed_tester.py` 的下载测速逻辑
- 移除 `validator.py` 中的阶段3（下载测速）
- 大幅减少 GitHub Actions 运行时间

#### 2. 改进延迟测试 (`scripts/clash_manager.py`)
测试 URL 改为被墙服务：
```python
TEST_URLS = [
    "https://www.google.com/generate_204",     # Google
    "https://www.youtube.com/generate_204",    # YouTube  
    "https://api.openai.com/v1/models",        # OpenAI (HEAD请求)
]
```

#### 3. 新增解锁能力测试 (`scripts/unlock_tester.py`)
测试多个被墙服务的连通性：
- Google 搜索
- YouTube
- OpenAI/ChatGPT
- GitHub (可选)
返回解锁评分 (0-100)

#### 4. 改进节点命名 (`scripts/validator.py`)
新命名格式（≤20字符）：
```
{国家码}{序号}_{解锁分}_{延迟}
例如：JP01_85_120  (日本节点，解锁85分，延迟120ms)
     HK12_92_45   (香港节点，解锁92分，延迟45ms)
     US03_78_180  (美国节点，解锁78分，延迟180ms)
```

#### 5. 改进评分机制 (`scripts/subscription_manager.py`)
新评分维度：
- **连通性 (40%)**：能否稳定连接被墙服务
- **解锁能力 (35%)**：能解锁多少服务
- **延迟 (15%)**：响应速度
- **稳定性 (10%)**：历史表现

### 文件改动清单
1. `scripts/unlock_tester.py` - 新增解锁能力测试模块
2. `scripts/clash_manager.py` - 延迟测试改用被墙服务
3. `scripts/validator.py` - 取消下载测速，集成解锁测试，改进命名
4. `scripts/subscription_manager.py` - 改进评分公式
5. `scripts/config.py` - 新增测试 URL 配置
6. `scripts/speed_tester.py` - 简化或删除
